# Cheung-Data 素材处理管线 v2 - 技术方案

## 概述

将 Cheung-Data 从"单一《魔幻手机》处理工具"升级为**多命名空间素材处理管线**，支持增量处理、匿名聚类、具体角色标注的完整流程。

---

## 命令行接口

```bash
# 1. 处理（ASR + 切分 + 聚类）
python process.py --namespace 魔幻手机                # 有 .npy 时自动标注角色
python process.py --namespace 魔幻手机 --no-label     # 不标注，只切分+聚类

# 2. 声纹注册（从 speaker_sources/ 生成 .npy）
python enroll.py --namespace 魔幻手机                 # 该空间下所有角色
python enroll.py --namespace 魔幻手机 --speaker 傻妞   # 单个角色

# 3. 重新标注（覆盖 speaker 字段 + 重命名切片）
python process.py --label 魔幻手机/EP01

# 4. 导出到 Agent
python export_for_agent.py --namespace 魔幻手机
```

典型使用顺序：

```
首次：process(--no-label) → enroll → --label → export
后续加集：process → export
修正声纹：往 speaker_sources 加素材 → enroll → --label → export
```

## 核心设计

### 输入规则

- 只处理 `wavs/{namespace}/` 下的 `.wav` 文件
- 不处理 MP4（人声分离由用户在外部用 UVR GUI 完成后放入 WAV）
- WAV 文件名即为该集/片段的标识，输出文件夹以此命名

### 增量处理

- 每次执行时扫描 `wavs/{namespace}/*.wav`
- 对每个 WAV，检查 `output/{namespace}/{wav_stem}/` 是否已存在
  - 已存在 → 跳过（打印 skip 信息）
  - 不存在 → 执行完整处理流程
- 如需重新处理：手动删除 `output/{namespace}/{wav_stem}/` 文件夹后重跑

## 执行流程

### process.py --namespace

```
扫描 wavs/{namespace}/*.wav
    │
    ├── output/{namespace}/{wav_stem}/ 已存在 → [跳过]
    │
    └── 不存在 → 开始处理：
        │
        ├── 步骤 1：语音识别 + 切分
        │   ├── WAV 转 16kHz mono（如有需要）
        │   ├── FunASR Paraformer-large 识别 → segments
        │   ├── 后处理纠错（按命名空间加载热词和纠错表）
        │   ├── 合并 VAD 碎片、过滤极短片段
        │   ├── 保存 segments.json（此时 cluster=null, speaker=null）
        │   └── FFmpeg 按时间戳切分为独立 WAV → clips/ 目录
        │       文件名：{序号}_{时间}.wav（临时命名）
        │
        ├── 步骤 2：匿名聚类
        │   ├── 对所有切片提取声纹 embedding
        │   ├── 层次聚类（AgglomerativeClustering）
        │   ├── 每个簇分配匿名标签：角色1、角色2、角色3...
        │   ├── 更新 segments.json 的 cluster 字段
        │   └── 重命名 clips/：{序号}_{时间}_{角色1}.wav
        │
        └── 步骤 3：角色标注（如有 .npy 且未指定 --no-label）
            ├── 加载 speaker_profiles/{namespace}/*.npy
            ├── 对每个 cluster 的平均 embedding 与已有角色比对
            ├── 匹配上 → speaker 字段填具体角色名
            ├── 匹配不上 → speaker 字段保持 cluster 值（角色1/角色2）
            ├── 更新 segments.json
            └── 重命名 clips/：{序号}_{时间}_{傻妞}.wav
```

### process.py --label

```
指定 output/{namespace}/{wav_stem}/ 目录
    │
    ├── 加载该目录的 segments.json
    ├── 加载 speaker_profiles/{namespace}/*.npy
    ├── 对每个 cluster 的平均 embedding 与角色声纹比对
    ├── 覆盖更新 segments.json 的 speaker 字段
    └── 重命名 clips/ 中的文件
```

### enroll.py

```
扫描 speaker_sources/{namespace}/{speaker_name}/*.wav
    │
    ├── 对每个 WAV 提取 embedding
    ├── 全部取平均 → 归一化
    ├── 保存为 speaker_profiles/{namespace}/{speaker_name}.npy
    └── 更新 speaker_profiles/{namespace}/profiles.json
```

## segments.json 格式

```json
[
  {
    "index": 1,
    "start": 5.2,
    "end": 8.1,
    "text": "小千哥哥你在干什么呢",
    "cluster": "角色1",
    "speaker": "傻妞",
    "score": 0.82,
    "file": "001_00m05s-00m08s_傻妞.wav"
  },
  {
    "index": 2,
    "start": 8.5,
    "end": 11.2,
    "text": "我在想办法回去",
    "cluster": "角色2",
    "speaker": null,
    "score": null,
    "file": "002_00m08s-00m11s_角色2.wav"
  }
]
```

字段说明：

| 字段 | 说明 |
|------|------|
| index | 序号（1-based） |
| start / end | 时间戳（秒） |
| text | 识别文本（可被用户手动修正） |
| cluster | 匿名聚类标签（步骤 2 产出，不变） |
| speaker | 具体角色名（标注后填入，初始 null） |
| score | 角色匹配置信度（标注后填入） |
| file | 当前切片文件名（随标注状态更新） |

## 切片文件命名规则

### 格式

```
{序号}_{起始时间}-{结束时间}_{角色}.wav
```

### 示例

```
001_00m05s-00m08s_傻妞.wav
002_00m08s-00m11s_陆小千.wav
003_00m12s-00m15s_角色3.wav
```

### 设计理由

- **序号在最前面**：资源管理器排列顺序天然正确
- **不把文本放文件名里**：文本可能很长、有特殊字符、修正后还得重命名。文本放 segments.json 里就够了
- **角色在最后**：方便肉眼浏览时快速识别是谁
- **程序通过 segments.json 定位文件**：不依赖解析文件名

## 匿名聚类算法

### 方案：层次聚类（Agglomerative Clustering）

```python
from sklearn.cluster import AgglomerativeClustering

def cluster_speakers(embeddings, threshold=0.3):
    """
    对所有切片的 embedding 做层次聚类
    
    threshold: 距离阈值（1 - cosine_similarity）
               0.3 意味着相似度 > 0.7 的会被归为同一人
    
    返回: labels 数组，每个切片对应一个簇编号
    """
    if len(embeddings) <= 1:
        return [0] * len(embeddings)
    
    clustering = AgglomerativeClustering(
        n_clusters=None,
        distance_threshold=threshold,
        metric="cosine",
        linkage="average",
    )
    labels = clustering.fit_predict(embeddings)
    return labels
```

### 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| metric | cosine | 声纹向量标准度量 |
| linkage | average | 簇间平均距离，比 single 更稳健 |
| distance_threshold | 0.30 | 初始值，相当于相似度 0.70 以上合并 |
| n_clusters | None | 不预设簇数，由阈值自动决定 |

### 聚类标签分配规则

- 按每个簇包含的片段数量降序排列
- 片段最多的簇标为"角色1"，次多的为"角色2"，以此类推
- 主角（说话最多）自然排在前面

## 具体角色标注逻辑

### 策略：按簇比对（非逐片段比对）

```
1. 对每个 cluster，取该簇所有切片的 embedding 平均值
2. 用簇平均 embedding 与 .npy 角色声纹做余弦相似度
3. 整簇统一标注（而非逐片段标注）
```

好处：
- 速度快（比对次数 = 簇数 × 角色数，远少于片段数 × 角色数）
- 更准确（平均 embedding 比单片段稳定）
- 逻辑清晰（一个簇 = 一个人 = 一个角色标签）

### 匹配规则

| 条件 | 结果 |
|------|------|
| 最高相似度 > 阈值 且 与第二名差距 > margin | 标注为该角色 |
| 最高相似度 > 阈值 但差距 ≤ margin | 保持 cluster 标签（不确定） |
| 最高相似度 ≤ 阈值 | 保持 cluster 标签（未知角色） |

## 声纹注册（enroll.py）

### 设计思路

声纹注册与处理流程完全解耦。用户将角色的 WAV 片段放入 `speaker_sources/{namespace}/{角色名}/`，运行 `enroll.py` 即可生成 .npy。

### 素材来源

- process.py 产出的切片（从 output/ 的 clips/ 中挑选确认是某角色的片段，复制到 speaker_sources/）
- 其他渠道获取的该角色音频片段

### 注册逻辑

1. 扫描 `speaker_sources/{namespace}/{speaker_name}/*.wav`
2. 对每个 WAV 提取 embedding（3D-Speaker ERes2NetV2）
3. 全部取平均 → 归一化 → 保存为 `speaker_profiles/{namespace}/{speaker_name}.npy`
4. 更新 `speaker_profiles/{namespace}/profiles.json`

### 特点

- **全量重算**：每次从 speaker_sources/ 全部素材重新计算平均 embedding，不是增量累加
- **素材可追加**：随时往文件夹里丢新 WAV，重跑 enroll.py 即可更新声纹
- **素材越多越准**：覆盖不同情绪、不同集数的样本，声纹更稳健

## 导出脚本：export_for_agent.py

### 定位

Data 和 Agent 之间的桥接工具。**只读** Data 的 output/，**生成** Agent 所需的 lines.json + 音频文件。

### 设计原则

- **只读不改**：不修改 Data 的 output/ 目录中的任何文件
- **全量重新生成**：每次导出都重新扫描并覆盖 Agent 端的 lines.json + audio/
- **只导出有效标注**：speaker 为 null 或匿名标签（"角色1"之类）的条目不导出
- **全局重编号**：多个 WAV 源文件的台词汇总后从 001 开始统一编号

### 导出逻辑

```
1. 扫描 output/{namespace}/ 下所有子目录的 segments.json
2. 筛选条件：
   - speaker 不为 null
   - speaker 不以"角色"开头（排除未匹配的匿名标签）
3. 按 WAV 源文件顺序 + index 顺序排列
4. 全局重编号：001, 002, 003...
5. 复制对应的切片 WAV 到 Agent 的 audio/ 目录，统一重命名为 {id}.wav
6. 生成 lines.json
```

### 产出文件

```
Cheung-Agent/
├── audio/
│   ├── 001.wav
│   ├── 002.wav
│   └── ...
├── lines.json
└── ...
```

### lines.json 格式

```json
[
  {
    "id": "001",
    "text": "小千哥哥你在干什么呢",
    "file": "001.wav",
    "speaker": "傻妞",
    "source": "EP01/003"
  }
]
```

| 字段 | 说明 |
|------|------|
| id | 全局唯一编号（字符串，3 位数字） |
| text | 台词文本 |
| file | 音频文件名（= id + .wav） |
| speaker | 角色名 |
| source | 溯源信息（{wav_stem}/{index}） |

## 与 v1 的主要变化

| 变化点 | v1 | v2 |
|--------|----|----|
| 输入格式 | MP4 + Vocals WAV | 仅 WAV（人声已分离） |
| 输入目录 | videos/ | wavs/{namespace}/ |
| 目录结构 | 扁平 | 命名空间隔离 |
| 处理模式 | 全量（清空重跑） | 增量（已存在则跳过） |
| 角色识别 | 逐片段比对 .npy | 先聚类再按簇比对 |
| 声纹注册 | 交互式标注（在处理流程中） | 独立脚本 enroll.py + 专用素材目录 |
| 声纹库 | 全局共享 | 按命名空间隔离 |
| 脚本数量 | 2 个 | 3 个（process + enroll + export） |
| 纠错/热词 | 硬编码 | config.json 按命名空间维护 |
| 切片命名 | [角色]_文本_序号_时间.wav | 序号_时间_角色.wav（不含文本） |
| 运行中交互 | 有（标注角色） | 无（全命令行参数控制） |

## 注意事项

1. **文件名不含文本**：切片文件名为 `{序号}_{时间}_{角色}.wav`，文本信息只存在于 segments.json。

2. **聚类结果不变**：cluster 字段一旦生成就不会被后续标注覆盖。它是"这个声音和谁是一组"的客观记录，speaker 才是"这个声音是谁"的标注结果。

3. **.npy 按命名空间隔离**：魔幻手机的声纹库和哆啦A梦完全独立，不会互相干扰。

4. **切片定位不依赖文件名**：通过 segments.json 的 file 字段定位切片文件。

5. **全量重算声纹**：enroll.py 每次从 speaker_sources/ 全部素材重新计算，不是增量累加。素材越多越准。

6. **无运行中交互**：所有行为由命令行参数决定，执行过程中不会停下来问用户。
