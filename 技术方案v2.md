# Cheung-Data 素材处理管线 v2 - 技术方案

## 概述

将 Cheung-Data 从"单一《魔幻手机》处理工具"升级为**多命名空间素材处理管线**，支持增量处理、匿名聚类、具体角色标注的完整流程。

---

## 命令行接口

```bash
# 1. 生成 edit.txt + reference.wav（ASR 识别 + 预切分）
python process.py --namespace 魔幻手机 --gen-edit 小千和傻妞对话.wav

# 2. 按 edit.txt 切分 + 聚类 + 标注
python process.py --namespace 魔幻手机 --apply-edit 小千和傻妞对话 --no-label   # 切分 + 聚类，不标注
python process.py --namespace 魔幻手机 --apply-edit 小千和傻妞对话             # 切分 + 聚类 + 有 .npy 时自动标注角色

# 3. 声纹注册（从 speaker_sources/ 生成 .npy）
python enroll.py --namespace 魔幻手机 --speaker 傻妞   # 单个角色
python enroll.py --namespace 魔幻手机                 # 该空间下所有角色

# 4. 重新标注（声纹库更新后，覆盖 speaker 字段 + 重命名切片）
python process.py --label 魔幻手机/小千和傻妞对话

# 5. 导出
python export.py --namespace 魔幻手机
```

```bash
# 1. 生成 edit.txt + reference.wav（ASR 识别 + 预切分）
python process.py --namespace 哆啦A梦 --gen-edit "第0592话 超级英雄".wav

# 2. 按 edit.txt 切分 + 聚类 + 标注
python process.py --namespace 哆啦A梦 --apply-edit "第0592话 超级英雄" --no-label   # 切分 + 聚类，不标注
python process.py --namespace 哆啦A梦 --apply-edit "第0592话 超级英雄"             # 切分 + 聚类 + 有 .npy 时自动标注角色

# 3. 声纹注册（从 speaker_sources/ 生成 .npy）
python enroll.py --namespace 哆啦A梦 --speaker 傻妞   # 单个角色
python enroll.py --namespace 哆啦A梦                 # 该空间下所有角色

# 4. 重新标注（声纹库更新后，覆盖 speaker 字段 + 重命名切片）
python process.py --label 哆啦A梦/"第0592话 超级英雄"

# 5. 导出
python export.py --namespace 哆啦A梦
```

典型使用顺序：

```
首次：--gen-edit EP01.wav → 人工编辑 → --apply-edit EP01 --no-label → 挑片段到 speaker_sources/ → enroll → --label → export
后续加集：--gen-edit EP02.wav → 人工编辑 → --apply-edit EP02 → export
修正声纹：往 speaker_sources 加素材 → enroll → --label → export
```

## 核心设计

### 输入规则

- 只处理 `wavs/{namespace}/` 下的 `.wav` 文件
- 不处理 MP4（人声分离由用户在外部用 UVR GUI 完成后放入 WAV）
- WAV 文件名即为该集/片段的标识，输出文件夹以此命名

### 增量处理

- 每次执行时扫描 `wavs/{namespace}/*.wav`
- 对每个 WAV，检查 `output/{namespace}/{wav_stem}/` 是否已存在
  - 已存在 → 跳过（打印 skip 信息）
  - 不存在 → 执行完整处理流程
- 如需重新处理：手动删除 `output/{namespace}/{wav_stem}/` 文件夹后重跑

## 执行流程

### process.py --gen-edit（生成可编辑文件）

```
输入：wavs/{namespace}/{filename}（指定具体 WAV 文件）
    │
    ├── output/{namespace}/{wav_stem}/edit.txt 已存在
    │   → 报错提示"edit.txt 已存在，请删除后重新执行"，退出
    │
    └── 不存在 → 开始处理：
        │
        ├── 步骤 1：语音识别
        │   ├── WAV 转 16kHz mono
        │   ├── FunASR Paraformer-large 识别 → 逗号级短句列表
        │   └── 后处理纠错
        │
        ├── 步骤 2：生成 edit.txt
        │   ├── 按句末标点（。？！；…）自动插入空行（预切分）
        │   ├── 每行格式：[{start}-{end}] {文本}
        │   └── 保存到 output/{namespace}/{wav_stem}/edit.txt
        │
        ├── 步骤 3：生成 reference.wav
        │   ├── 将 ASR 切好的逗号级短句音频按顺序拼接
        │   ├── 每个片段与 edit.txt 每行一一对应
        │   ├── 片段之间插入 2 秒静音
        │   └── 拼接保存为 output/{namespace}/{wav_stem}/reference.wav
        │
        └── 步骤 4：自动打开
            ├── Notepad++ 打开 edit.txt
            └── 系统默认播放器打开 reference.wav
```

### process.py --apply-edit（按编辑结果切分 + 聚类 + 标注）

```
输入：output/{namespace}/{episode_name}/edit.txt（指定具体 episode）
    │
    ├── edit.txt 不存在 → 报错退出
    │
    └── 存在 → 开始处理：
        │
        ├── 步骤 1：解析 edit.txt
        │   ├── 按空行分组
        │   ├── 每组 = 一个切片
        │   ├── 起始时间 = 组内第一行的 start
        │   ├── 结束时间 = 组内最后一行的 end
        │   └── 文本 = 组内各行文本拼接
        │
        ├── 步骤 2：切分音频
        │   ├── 清空并重建 clips/ 目录
        │   ├── FFmpeg 从源 WAV 按时间范围切分
        │   └── 文件名：{序号}_{时间}.wav
        │
        ├── 步骤 3：匿名聚类
        │   ├── 对所有切片提取声纹 embedding
        │   ├── 层次聚类（AgglomerativeClustering）
        │   ├── 每个簇分配匿名标签：角色1、角色2、角色3...
        │   ├── 更新 segments.json 的 cluster 字段
        │   └── 重命名 clips/：{序号}_{时间}_{角色1}.wav
        │
        ├── 步骤 4：角色标注（如有 .npy 且未指定 --no-label）
        │   ├── 加载 speaker_profiles/{namespace}/*.npy
        │   ├── 对每个 cluster 的平均 embedding 与角色声纹比对
        │   ├── 匹配上 → speaker 字段填角色名
        │   ├── 匹配不上 → speaker 保持 null，文件名用 cluster 标签
        │   └── 重命名 clips/：{序号}_{时间}_{傻妞}.wav
        │
        └── 步骤 5：保存 segments.json
            └── 不删除 edit.txt 和 reference.wav
```

### process.py --label

```
指定 output/{namespace}/{wav_stem}/ 目录
    │
    ├── 加载该目录的 segments.json
    ├── 加载 speaker_profiles/{namespace}/*.npy
    ├── 对每个 cluster 的平均 embedding 与角色声纹比对
    ├── 覆盖更新 segments.json 的 speaker 字段
    └── 重命名 clips/ 中的文件
```

### enroll.py

```
扫描 speaker_sources/{namespace}/{speaker_name}/*.wav
    │
    ├── 对每个 WAV 提取 embedding
    ├── 全部取平均 → 归一化
    ├── 保存为 speaker_profiles/{namespace}/{speaker_name}.npy
    └── 更新 speaker_profiles/{namespace}/profiles.json
```

## segments.json 格式

```json
[
  {
    "index": 1,
    "start": 5.2,
    "end": 8.1,
    "text": "小千哥哥你在干什么呢",
    "cluster": "角色1",
    "speaker": "傻妞",
    "score": 0.82,
    "file": "001_00m05s-00m08s_傻妞.wav"
  },
  {
    "index": 2,
    "start": 8.5,
    "end": 11.2,
    "text": "我在想办法回去",
    "cluster": "角色2",
    "speaker": null,
    "score": null,
    "file": "002_00m08s-00m11s_角色2.wav"
  }
]
```

字段说明：

| 字段 | 说明 |
|------|------|
| index | 序号（1-based） |
| start / end | 时间戳（秒） |
| text | 识别文本（可被用户手动修正） |
| cluster | 匿名聚类标签（步骤 2 产出，不变） |
| speaker | 具体角色名（标注后填入，初始 null） |
| score | 角色匹配置信度（标注后填入） |
| file | 当前切片文件名（随标注状态更新） |

## 切片文件命名规则

### 格式

```
{序号}_{起始时间}-{结束时间}_{角色}.wav
```

### 示例

```
001_00m05s-00m08s_傻妞.wav
002_00m08s-00m11s_陆小千.wav
003_00m12s-00m15s_角色3.wav
```

### 设计理由

- **序号在最前面**：资源管理器排列顺序天然正确
- **不把文本放文件名里**：文本可能很长、有特殊字符、修正后还得重命名。文本放 segments.json 里就够了
- **角色在最后**：方便肉眼浏览时快速识别是谁
- **程序通过 segments.json 定位文件**：不依赖解析文件名

## 匿名聚类算法

### 方案：层次聚类（Agglomerative Clustering）

```python
from sklearn.cluster import AgglomerativeClustering

def cluster_speakers(embeddings, threshold=0.3):
    """
    对所有切片的 embedding 做层次聚类
    
    threshold: 距离阈值（1 - cosine_similarity）
               0.3 意味着相似度 > 0.7 的会被归为同一人
    
    返回: labels 数组，每个切片对应一个簇编号
    """
    if len(embeddings) <= 1:
        return [0] * len(embeddings)
    
    clustering = AgglomerativeClustering(
        n_clusters=None,
        distance_threshold=threshold,
        metric="cosine",
        linkage="average",
    )
    labels = clustering.fit_predict(embeddings)
    return labels
```

### 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| metric | cosine | 声纹向量标准度量 |
| linkage | average | 簇间平均距离，比 single 更稳健 |
| distance_threshold | 0.30 | 初始值，相当于相似度 0.70 以上合并 |
| n_clusters | None | 不预设簇数，由阈值自动决定 |

### 聚类标签分配规则

- 按每个簇包含的片段数量降序排列
- 片段最多的簇标为"角色1"，次多的为"角色2"，以此类推
- 主角（说话最多）自然排在前面

## 具体角色标注逻辑

### 策略：按簇比对（非逐片段比对）

```
1. 对每个 cluster，取该簇所有切片的 embedding 平均值
2. 用簇平均 embedding 与 .npy 角色声纹做余弦相似度
3. 整簇统一标注（而非逐片段标注）
```

好处：
- 速度快（比对次数 = 簇数 × 角色数，远少于片段数 × 角色数）
- 更准确（平均 embedding 比单片段稳定）
- 逻辑清晰（一个簇 = 一个人 = 一个角色标签）

### 匹配规则

| 条件 | 结果 |
|------|------|
| 最高相似度 > 阈值 且 与第二名差距 > margin | speaker 填该角色名 |
| 最高相似度 > 阈值 但差距 ≤ margin | speaker 保持 null（不确定，文件名用 cluster 标签） |
| 最高相似度 ≤ 阈值 | speaker 保持 null（未匹配，文件名用 cluster 标签） |

## 声纹注册（enroll.py）

### 设计思路

声纹注册与处理流程完全解耦。用户将角色的 WAV 片段放入 `speaker_sources/{namespace}/{角色名}/`，运行 `enroll.py` 即可生成 .npy。

### 素材来源

- process.py 产出的切片（从 output/ 的 clips/ 中挑选确认是某角色的片段，复制到 speaker_sources/）
- 其他渠道获取的该角色音频片段

### 注册逻辑

1. 扫描 `speaker_sources/{namespace}/{speaker_name}/*.wav`
2. 对每个 WAV 提取 embedding（3D-Speaker ERes2NetV2）
3. 全部取平均 → 归一化 → 保存为 `speaker_profiles/{namespace}/{speaker_name}.npy`
4. 更新 `speaker_profiles/{namespace}/profiles.json`

### 特点

- **全量重算**：每次从 speaker_sources/ 全部素材重新计算平均 embedding，不是增量累加
- **素材可追加**：随时往文件夹里丢新 WAV，重跑 enroll.py 即可更新声纹
- **素材越多越准**：覆盖不同情绪、不同集数的样本，声纹更稳健

## 导出脚本：export.py

### 定位

将处理结果按角色组织导出。**只读** output/，**生成**按命名空间和角色分组的音频文件 + lines.json。

### 设计原则

- **只读不改**：不修改 Data 的 output/ 目录中的任何文件
- **全量重新生成**：每次导出先清空 Agent 端对应目录，再重新写入
- **只导出有效标注**：speaker 为 null 或匿名标签（"角色1"之类）的条目不导出
- **按命名空间 + 角色名组织**：每个角色独立文件夹，各自编号
- **幂等**：多次运行结果一致，不会残留旧文件

### 导出逻辑

```
1. 扫描 output/{namespace}/ 下所有子目录的 segments.json
2. 筛选条件：
   - speaker 不为 null
   - speaker 不以"角色"开头（排除未匹配的匿名标签）
3. 按 speaker 分组，每组内按 WAV 源文件顺序 + index 排列
4. 每个角色独立编号：001, 002, 003...
5. 清空 export/{namespace}/{speaker}/ 目录
6. 复制切片 WAV → {id}.wav
7. 生成每个角色的 lines.json
```

## 与 v1 的主要变化

| 变化点 | v1 | v2 |
|--------|----|----|
| 输入格式 | MP4 + Vocals WAV | 仅 WAV（人声已分离） |
| 输入目录 | videos/ | wavs/{namespace}/ |
| 目录结构 | 扁平 | 命名空间隔离 |
| 处理模式 | 全量（清空重跑） | 增量（已存在则跳过） |
| 切分方式 | 纯自动（ASR 句子边界） | 人工辅助（--gen-edit → 编辑 → --apply-edit） |
| 角色识别 | 逐片段比对 .npy | 先聚类再按簇比对 |
| 声纹注册 | 交互式标注（在处理流程中） | 独立脚本 enroll.py + 专用素材目录 |
| 声纹库 | 全局共享 | 按命名空间隔离 |
| 脚本数量 | 2 个 | 3 个（process + enroll + export） |
| 纠错/热词 | 硬编码 | config.json 按命名空间维护 |
| 切片命名 | [角色]_文本_序号_时间.wav | 序号_时间_角色.wav（不含文本） |
| 运行中交互 | 有（标注角色） | 无（全命令行参数控制） |

---

## 注意事项

1. **文件名不含文本**：切片文件名为 `{序号}_{时间}_{角色}.wav`，文本信息只存在于 segments.json。

2. **聚类结果不变**：cluster 字段一旦生成就不会被后续标注覆盖。它是"这个声音和谁是一组"的客观记录，speaker 才是"这个声音是谁"的标注结果。

3. **.npy 按命名空间隔离**：魔幻手机的声纹库和哆啦A梦完全独立，不会互相干扰。

4. **切片定位不依赖文件名**：通过 segments.json 的 file 字段定位切片文件。

5. **全量重算声纹**：enroll.py 每次从 speaker_sources/ 全部素材重新计算，不是增量累加。素材越多越准。

6. **无运行中交互**：所有行为由命令行参数决定，执行过程中不会停下来问用户。

### 常见操作

| 操作 | 做法 | 对 export 的影响 |

## 人工辅助切分方案

### 问题背景

自动切分依赖 ASR 的句子边界（标点）。当两人对话间隔极短时，VAD 不会切开，ASR 把两人的话识别为一句，导致切片混入多人声音。这是 VAD + 标点模型的设计上限，无法纯自动解决。

### 方案思路

将"切分判断"交给人，将"切分执行"交给脚本。人只需在文本文件中打回车标记切分点，脚本自动按时间戳从源 WAV 切出音频。

### 流程

```
第一步（自动）：--gen-edit EP01.wav
    ├── ASR 识别，按逗号级粒度输出短句列表 + 时间戳
    ├── 自动按句末标点（。？！）预切分（已插好空行）
    ├── 生成 edit.txt
    ├── 将 ASR 切好的短句音频按顺序拼接（片段间 2s 静音）→ reference.wav
    ├── 自动用 Notepad++ 打开 edit.txt
    └── 自动用系统默认播放器打开 reference.wav

第二步（人工）：
    ├── 对照 reference.wav 听音频（每行对应一个片段，2s 静音 = 换行）
    ├── 在 edit.txt 中调整切分点（增删空行）
    ├── 删除不要的行
    ├── 修正 ASR 识别错误的文字
    └── 保存

第三步（自动）：--apply-edit EP01
    ├── 读取编辑后的 edit.txt，按空行分组
    ├── 每组的起始时间到结束时间 = 一个切片
    ├── FFmpeg 从源 WAV 切分 → clips/
    ├── 聚类（角色1、角色2...）
    ├── 生成 segments.json
    └── 如有声纹库且未 --no-label → 标注具体角色
```

### edit.txt 格式

自动生成的文件（已按句末标点预切分，空行已插好）：

```
[00m03s-00m05s] 我知道了。

[00m05s-00m08s] 那我就让你当超级英雄啊，
[00m08s-00m11s] 哆啦a梦你真不愧是我的好朋友。

[00m12s-00m15s] 拜托居然跟胖虎说的一模一样，
[00m15s-00m18s] 你太得意忘形了。
```

脚本已根据句末标点自动插入空行，用户不需要从头逐行回车。大部分情况只需微调：
- 某些地方空行多余 → 删掉空行（合并为一个切片）
- 某些地方漏切 → 插入空行
- 某行不想要 → 直接删除该行
- 文字有误 → 直接修改

人工编辑后的示例：

```
[00m05s-00m08s] 那我就让你当超级英雄啊，

[00m08s-00m11s] 哆啦A梦你真不愧是我的好朋友，
[00m12s-00m15s] 拜托居然跟胖虎说的一模一样，
[00m15s-00m18s] 你太得意忘形了。
```

说明：
- 删除了第 1 行 → 该时间段不生成切片
- 把原来第 2、3 行之间的空行删了 → 合并为一个切片
- 在第 2 行后保留空行 → 此处切断
- 把"哆啦a梦"改成了"哆啦A梦" → 最终文本使用修正后的版本

### reference.wav

`--gen-edit` 同时生成一个 `reference.wav` 参考音频：

- 将 ASR 切好的逗号级短句音频按顺序拼接
- 每个片段与 edit.txt 的每一行**一一对应**
- 片段之间插入 **2 秒静音**，听到静音就知道换行了
- 配合 edit.txt 使用：数静音次数即可定位到 edit.txt 的对应行

用途：用户边听 reference.wav 边看 edit.txt，判断哪些地方需要合并（同一人连续说话）或拆分（两人对话混在一起）。

### 切分逻辑

- 连续的行（无空行间隔）归为同一个切片
- 空行 = 切分点
- 每个切片的时间范围 = 第一行的 start 到最后一行的 end
- 每个切片的文本 = 各行文本拼接
- 从源 WAV 按时间范围切出独立音频文件
- **删除行 = 不要该段**：脚本只处理文件中剩余的行，被删除的行对应的时间段不会生成切片
- **文本可修正**：方括号内的时间戳用于定位切分，方括号后的文字作为最终文本。直接修改文字即可纠正 ASR 错误

### --apply-edit 后的角色标注

`--apply-edit` 的后处理流程：**先聚类，再标注**。

聚类的作用：
- 把同一个人的片段归到一组（角色1、角色2...）
- 方便用户按组批量挑选片段到 `speaker_sources/` 做声纹注册
- 不需要逐片段听，直接把某一组复制过去即可

标注逻辑：
- 如果 `speaker_profiles/{namespace}/` 下有 .npy 声纹文件 → 按簇平均 embedding 与角色声纹比对
- 如果没有声纹库或指定了 `--no-label` → 只聚类不标注，segments.json 中 speaker 保持 null

```bash
python process.py --namespace 魔幻手机 --apply-edit EP01 --no-label   # 切分 + 聚类（不标注）
python process.py --namespace 魔幻手机 --apply-edit EP01              # 切分 + 聚类 + 标注角色
```

### 增量/覆盖规则

- `--gen-edit`：如果 `output/{namespace}/{wav_stem}/edit.txt` 已存在 → **报错提示"edit.txt 已存在，请删除后重新执行"**，退出（保护人工编辑成果）
- `--apply-edit`：如果 `output/{namespace}/{episode}/edit.txt` 不存在 → 报错退出
- `--apply-edit` 会**重新生成** clips/ 和 segments.json（覆盖旧的切分结果）
- `--apply-edit` **不会删除** edit.txt 和 reference.wav

### 自动打开编辑器和播放器

`--gen-edit` 执行完毕后自动：

1. 用 **Notepad++** 打开 edit.txt（路径：`C:\Program Files\Notepad++\notepad++.exe`）
2. 用 **系统默认播放器** 打开 reference.wav（通过 `os.startfile()`）

如果 Notepad++ 未安装或路径不对，fallback 到系统默认的 .txt 关联程序打开。

> 注：只打开最后一个处理的文件。如果一次性处理了多个 WAV，前面的只生成文件不打开，最后一个自动打开。

### 命令行接口

```bash
# 第一步：生成 edit.txt + reference.wav，自动打开编辑器和播放器
python process.py --namespace 魔幻手机 --gen-edit EP01.wav

# 第二步：人工编辑 output/魔幻手机/EP01/edit.txt

# 第三步：按编辑结果切分 + 聚类 + 标注
python process.py --namespace 魔幻手机 --apply-edit EP01
python process.py --namespace 魔幻手机 --apply-edit EP01 --no-label   # 不标注
```

### 优点

- 切分精度 100%（人在判断切分点）
- **脚本自动按标点预切分**，大部分情况只需微调而非从头回车
- **reference.wav 参考音频**方便对照听取
- 自动打开 Notepad++ 和播放器，减少手动操作
- 一站式审核：切分、排除、纠错在同一个文件完成
- 文本和音频天然对应（每个切片的文本就是对应行拼接）
- 与现有自动模式兼容，不影响已有的全自动流程

### 适用场景

- 对切分质量要求高的素材（训练数据等）
- 混音问题多的音频（多人快速对话）
- 首次处理新命名空间时，手动把控一遍质量
