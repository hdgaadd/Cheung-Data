# Cheung-Data 素材处理管线 v2 - 技术方案

## 概述

将 Cheung-Data 从"单一《魔幻手机》处理工具"升级为**多命名空间素材处理管线**，支持增量处理、匿名聚类、具体角色标注的完整流程。

### 关于 config.json

把原来硬编码在代码里的热词（HOTWORDS）和纠错表（CORRECTIONS）抽出来，按命名空间配置。加新命名空间时只改 JSON 不改代码。

### 关于 process.py

把原来的 `extract_dialogue.py` + `extract_with_speaker.py` 合并为一个统一入口。新流程是"切分 → 聚类 → 标注"一条龙，一个脚本搞定。

## 核心设计

### 输入规则

- 只处理 `wavs/{namespace}/` 下的 `.wav` 文件
- 不处理 MP4（人声分离由用户在外部用 UVR GUI 完成后放入 WAV）
- WAV 文件名即为该集/片段的标识，输出文件夹以此命名

### 增量处理

- 每次执行时扫描所有 `wavs/{namespace}/*.wav`
- 对每个 WAV，检查 `output/{namespace}/{wav_stem}/` 是否已存在
  - 已存在 → 跳过（打印 skip 信息）
  - 不存在 → 执行完整处理流程
- 如需重新处理：手动删除 `output/{namespace}/{wav_stem}/` 文件夹后重跑

## 执行流程

### 完整流程图

```
扫描 wavs/{namespace}/*.wav
    │
    ├── output/{namespace}/{wav_stem}/ 已存在 → [跳过]
    │
    └── 不存在 → 开始处理：
        │
        ├── 步骤 1：语音识别 + 切分
        │   ├── WAV 转 16kHz mono（如有需要）
        │   ├── FunASR Paraformer-large 识别 → segments
        │   ├── 后处理纠错（按命名空间加载热词和纠错表）
        │   ├── 合并 VAD 碎片、过滤极短片段
        │   ├── 保存 segments.json（此时 cluster=null, speaker=null）
        │   └── FFmpeg 按时间戳切分为独立 WAV → clips/ 目录
        │       文件名：{序号}_{时间}.wav（临时命名）
        │
        ├── 步骤 2：匿名聚类
        │   ├── 对所有切片提取声纹 embedding
        │   ├── 层次聚类（AgglomerativeClustering）
        │   ├── 每个簇分配匿名标签：角色1、角色2、角色3...
        │   ├── 更新 segments.json 的 cluster 字段
        │   └── 重命名 clips/：{序号}_{时间}_{角色1}.wav
        │
        └── 步骤 3：提示用户选择
            │
            ├── (1) 根据已有 .npy 进行具体角色标注
            │   ├── 加载 speaker_profiles/{namespace}/*.npy
            │   ├── 对每个 cluster 的平均 embedding 与已有角色比对
            │   ├── 匹配上 → speaker 字段填具体角色名
            │   ├── 匹配不上 → speaker 字段保持 cluster 值（角色1/角色2）
            │   ├── 更新 segments.json
            │   └── 重命名 clips/：{序号}_{时间}_{傻妞}.wav
            │
            ├── (2) 生成/更新该命名空间的 .npy（交互式注册）
            │   ├── 展示当前切片列表（按 cluster 分组显示）
            │   ├── 用户为每个 cluster 指定角色名
            │   ├── 取该 cluster 的所有切片 embedding 做平均
            │   └── 保存到 speaker_profiles/{namespace}/{角色名}.npy
            │
            └── (3) 跳过，稍后处理
                └── 保持当前匿名标签状态，后续用 --label 单独执行标注
```

## 命令行接口

```bash
# 默认模式：扫描所有命名空间，增量处理新 WAV
python process.py

# 指定命名空间（只扫描该命名空间下的 WAV）
python process.py --namespace 魔幻手机

# 对已处理但未标注角色的目录单独执行标注
python process.py --label 魔幻手机/EP01
```

只有三个用法，保持简洁：
- 正常跑：`python process.py`（增量，已存在的自动跳过）
- 限定范围：`python process.py --namespace 魔幻手机`
- 单独标注：`python process.py --label 魔幻手机/EP01`

想重新处理某个 WAV → 手动删掉 `output/魔幻手机/EP01/` 文件夹 → 再跑 `python process.py`

## segments.json 格式

```json
[
  {
    "index": 1,
    "start": 5.2,
    "end": 8.1,
    "text": "小千哥哥你在干什么呢",
    "cluster": "角色1",
    "speaker": "傻妞",
    "score": 0.82,
    "file": "001_00m05s-00m08s_傻妞.wav"
  },
  {
    "index": 2,
    "start": 8.5,
    "end": 11.2,
    "text": "我在想办法回去",
    "cluster": "角色2",
    "speaker": null,
    "score": null,
    "file": "002_00m08s-00m11s_角色2.wav"
  }
]
```

字段说明：

| 字段 | 说明 |
|------|------|
| index | 序号（1-based） |
| start / end | 时间戳（秒） |
| text | 识别文本（可被用户手动修正） |
| cluster | 匿名聚类标签（步骤 2 产出，不变） |
| speaker | 具体角色名（步骤 3 标注后填入，初始 null） |
| score | 角色匹配置信度（步骤 3 产出） |
| file | 当前切片文件名（随标注状态更新） |

## 切片文件命名规则

### 格式

```
{序号}_{起始时间}-{结束时间}_{角色}.wav
```

### 示例

```
001_00m05s-00m08s_傻妞.wav
002_00m08s-00m11s_陆小千.wav
003_00m12s-00m15s_角色3.wav
```

### 设计理由

- **序号在最前面**：资源管理器排列顺序天然正确
- **不把文本放文件名里**：文本可能很长、有特殊字符、修正后还得重命名。文本放 segments.json 里就够了，Agent 从 JSON 读文本
- **角色在最后**：方便肉眼浏览时快速识别是谁
- **程序通过 segments.json 定位文件**：不依赖解析文件名

### Cheung-Agent 如何使用

Agent 通过读取 segments.json 获得 `text` + `file` + `speaker` 的映射关系，不需要从文件名里解析文本。

## 匿名聚类算法

### 方案：层次聚类（Agglomerative Clustering）

```python
from sklearn.cluster import AgglomerativeClustering

def cluster_speakers(embeddings, threshold=0.3):
    """
    对所有切片的 embedding 做层次聚类
    
    threshold: 距离阈值（1 - cosine_similarity）
               0.3 意味着相似度 > 0.7 的会被归为同一人
    
    返回: labels 数组，每个切片对应一个簇编号
    """
    if len(embeddings) <= 1:
        return [0] * len(embeddings)
    
    clustering = AgglomerativeClustering(
        n_clusters=None,
        distance_threshold=threshold,
        metric="cosine",
        linkage="average",
    )
    labels = clustering.fit_predict(embeddings)
    return labels
```

### 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| metric | cosine | 声纹向量标准度量 |
| linkage | average | 簇间平均距离，比 single 更稳健 |
| distance_threshold | 0.30 | 初始值，相当于相似度 0.70 以上合并 |
| n_clusters | None | 不预设簇数，由阈值自动决定 |

### 聚类标签分配规则

- 按每个簇包含的片段数量降序排列
- 片段最多的簇标为"角色1"，次多的为"角色2"，以此类推
- 主角（说话最多）自然排在前面

## 具体角色标注逻辑

### 策略：按簇比对（非逐片段比对）

```
1. 对每个 cluster，取该簇所有切片的 embedding 平均值
2. 用簇平均 embedding 与 .npy 角色声纹做余弦相似度
3. 整簇统一标注（而非逐片段标注）
```

好处：
- 速度快（比对次数 = 簇数 × 角色数，远少于片段数 × 角色数）
- 更准确（平均 embedding 比单片段稳定）
- 逻辑清晰（一个簇 = 一个人 = 一个角色标签）

### 匹配规则

| 条件 | 结果 |
|------|------|
| 最高相似度 > 阈值 且 与第二名差距 > margin | 标注为该角色 |
| 最高相似度 > 阈值 但差距 ≤ margin | 保持 cluster 标签（不确定） |
| 最高相似度 ≤ 阈值 | 保持 cluster 标签（未知角色） |

## 用户操作流程

### 首次使用新命名空间

```
1. 在 wavs/ 下创建文件夹，如 wavs/魔幻手机/
2. 把 UVR 分离好的人声 WAV 放入
3. 在 config.json 中添加该命名空间的热词和纠错表
4. 执行 python process.py
5. 程序处理完后提示选择 → 选 (2) 生成 .npy

7. 完成
```

### 后续增量处理

```
1. 新 WAV 放入 wavs/魔幻手机/
2. 执行 python process.py
3. 程序自动跳过已处理的，只处理新 WAV
4. 切分 + 聚类完成后提示选择 → 选 (1) 用已有 .npy 标注
5. 完成
```

### 修正文本后重新标注

```
1. 手动编辑 output/魔幻手机/EP01/segments.json 中的 text 字段
   （或用 AI 辅助批量修正）
2. 执行 python process.py --label 魔幻手机/EP01
3. 程序根据 segments.json 重新命名所有切片文件（用最新的 text 和 speaker）
```

### 重新处理某个 WAV

```
1. 删除 output/魔幻手机/EP01/ 文件夹
2. 执行 python process.py
3. 程序发现该 WAV 没有对应输出，重新处理
```

## 导出脚本：export_for_agent.py

### 定位

Data 和 Agent 之间的桥接工具。**只读** Data 的 output/，**生成** Agent 所需的 lines.json + 音频文件。

### 设计原则

- **只读不改**：不修改 Data 的 output/ 目录中的任何文件（不重命名、不删除、不修改 segments.json）
- **全量重新生成**：每次导出都重新扫描并覆盖 Agent 端的 lines.json + audio/，保证是最新快照
- **只导出有效标注**：speaker 为 null 或匿名标签（"角色1"之类）的条目不导出
- **全局重编号**：多个 WAV 源文件的台词汇总后从 001 开始统一编号，不会重复

### 命令行

```bash
# 导出指定命名空间的所有有效标注台词到 Agent
python export_for_agent.py --namespace 魔幻手机
```

- 输出路径固定在 config.json 中配置（`agent_path` 字段，默认 `../Cheung-Agent/`）
- 一个命名空间的所有角色台词全部导出（Agent 需要所有角色的台词库）
- 纯读取 + 复制，几百条几秒搞定

### 导出逻辑

```
1. 扫描 output/{namespace}/ 下所有子目录的 segments.json
2. 筛选条件：
   - speaker 不为 null
   - speaker 不以"角色"开头（排除未匹配的匿名标签）
   - 如指定了 --speaker，只取该角色
3. 按 WAV 源文件顺序 + index 顺序排列
4. 全局重编号：001, 002, 003...
5. 复制对应的切片 WAV 到 Agent 的 audio/ 目录，统一重命名为 {id}.wav
6. 生成 lines.json
```

### 产出文件

Agent 端目录变化：

```
Cheung-Agent/
├── audio/
│   ├── 001.wav      ← 从 Data 复制并重编号
│   ├── 002.wav
│   ├── 003.wav
│   └── ...
├── lines.json       ← 导出脚本生成
└── ...（其他 Agent 文件不动）
```

### lines.json 导出格式

```json
[
  {
    "id": "001",
    "text": "小千哥哥你在干什么呢",
    "file": "001.wav",
    "speaker": "傻妞",
    "source": "EP01/003"
  },
  {
    "id": "002",
    "text": "你别着急让傻妞帮你分析",
    "file": "002.wav",
    "speaker": "傻妞",
    "source": "EP01/007"
  },
  {
    "id": "003",
    "text": "从这个角度来看并不是坏事",
    "file": "003.wav",
    "speaker": "傻妞",
    "source": "EP02/001"
  }
]
```

字段说明：

| 字段 | 说明 |
|------|------|
| id | 全局唯一编号（字符串，3 位数字），Agent 用此 id 做 [PLAY:xxx] 匹配 |
| text | 台词文本（来自 segments.json，可能被手动修正过） |
| file | 音频文件名（= id + .wav） |
| speaker | 角色名（方便 Agent 按角色筛选，当前阶段塞 prompt 时可选是否显示） |
| source | 溯源信息（{wav_stem}/{index}），方便调试定位原始切片，Agent 运行时不用 |

### Agent 端兼容性

导出后 Agent 不需要改任何代码：
- lines.json 格式与当前完全兼容（id + text + file）
- 只是多了 speaker 和 source 两个字段，Agent 读取时忽略即可
- 音频文件统一为 WAV 格式，Agent 的 pygame / HTML5 audio 都支持 WAV 播放

### 数据流向

```
Data（output/）─── 只读 ───→ export_for_agent.py ─── 生成 ───→ Agent（audio/ + lines.json）
      ↑                                                              ↑
  数据源，不被修改                                             每次导出全量覆盖
```

单向数据流：Data 产出 → 导出脚本只读转换 → Agent 消费。

## 注意事项

1. **文件名不含文本**：切片文件名为 `{序号}_{时间}_{角色}.wav`，文本信息只存在于 segments.json。Agent 从 JSON 读取 text 字段，不解析文件名。

2. **聚类结果不变**：cluster 字段一旦生成就不会被后续标注覆盖。它是"这个声音和谁是一组"的客观记录，speaker 才是"这个声音是谁"的标注结果。

3. **.npy 按命名空间隔离**：魔幻手机的声纹库和哆啦A梦完全独立，不会互相干扰。

4. **切片定位不依赖文件名**：通过 segments.json 的 index + 时间戳定位切片文件，而非解析文件名。即使文件名被手动改过，程序仍能正确操作（通过扫描 clips/ 目录匹配时间戳后缀）。

5. **向后兼容**：旧的 output 目录结构（v1 格式）不会被自动迁移，建议手动整理或在新目录下重新处理。
