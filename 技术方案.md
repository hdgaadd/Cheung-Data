## 实现范围

### 当前版本（基础版）

1. 扫描 videos/ 目录下的 .mp4 文件及对应的人声 WAV 文件
2. 优先使用 UVR GUI 手动生成的人声 WAV；若无则 FFmpeg 提取原始音频
3. FunASR Paraformer-large 识别中文对话，输出带强制对齐时间戳的 segments
4. 汇总对话文本 → 保存为纯文本（一行一句）
5. 按 segment 时间戳用 FFmpeg 切分为独立 WAV 文件
6. 输出到 output/ 目录
7. 删除临时音频文件

### 角色识别版（纯 Embedding 比对）

- 基于 3D-Speaker ERes2NetV2 模型提取声纹向量
- 直接对切分片段做 embedding 比对，不跑 diarization
- 详见下方「声纹角色识别技术方案」
- 仓库地址：https://github.com/modelscope/3D-Speaker

## 核心技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 语音识别 | FunASR Paraformer-large | 中文语音识别，强制对齐时间戳，字级精度 |
| 人声分离 | Ultimate Vocal Remover (GUI) | 手动操作，使用 MDX-Net 模型去除背景音乐 |
| 声纹提取 | 3D-Speaker ERes2NetV2 | 192 维声纹向量，中文说话人训练，余弦相似度比对 |
| 音频提取 | FFmpeg | 从 MP4 提取音频轨道（作为无人声文件时的 fallback） |
| 音频切分 | FFmpeg | 按时间戳精确切割为独立 WAV 文件 |
| 编排 | Python 脚本 | 串联整个流程 |

### 为什么选 FunASR 替代 faster-whisper

| 对比项 | faster-whisper (large-v3) | FunASR (Paraformer-large) |
|--------|--------------------------|---------------------------|
| 中文识别精度 | 好 | 更好（中文专项优化） |
| 时间戳方式 | 估算（基于注意力权重） | 强制对齐（基于 CTC/对齐模型） |
| 尾音截断问题 | 常见（"了"字等被截掉） | 基本不存在（对齐精度高） |
| HuggingFace Token | 不需要 | 不需要 |
| 模型来源 | HuggingFace | ModelScope（国内下载快） |
| VAD | 内置简单 VAD | 内置 FSMN-VAD（效果更好） |
| 热词支持 | 通过 initial_prompt 间接引导 | 原生热词列表，精确度更高 |

### 为什么选 3D-Speaker 替代 pyannote

| 对比项 | pyannote/embedding | 3D-Speaker ERes2NetV2 |
|--------|-------------------|----------------------|
| 训练数据 | 英文/欧洲语言为主 | 中文说话人数据为主 |
| 适合场景 | 长音频聚类（diarization） | 已知角色集合、逐片段认人 |
| Embedding 维度 | 512 维 | 192 维 |
| 跨集复用 | 需要 HF Token | 无需 Token，ModelScope 直接下载 |
| 中文人声区分度 | 一般（阈值需压低到 0.4+） | 好（阈值可回到 0.6-0.7 正常区间） |
| 依赖 | pyannote.audio + torch | modelscope + 3D-Speaker |

## 音频预处理：人声分离

### 方案说明

人声分离**不在代码中自动执行**，由用户手动使用 Ultimate Vocal Remover (UVR) GUI 完成。

原因：
1. UVR GUI 操作简单，可视化调参方便
2. 人声分离只需每集做一次，不是高频操作
3. 避免引入 audio-separator 的复杂依赖和显存占用
4. GUI 版可以实时试听效果，确认质量后再跑后续流程

### 操作流程

```
1. 打开 Ultimate Vocal Remover GUI
2. 选择模型：UVR-MDX-NET-Inst_HQ_3
3. 输入：videos/ 下的 MP4 文件（UVR 可直接处理视频）
4. 输出目录：设为 videos/（和原视频放一起）
5. 点击分离 → 生成 xxx_(Vocals).wav
6. 运行 extract_dialogue.py，代码会自动检测并使用人声 WAV
```

### 代码检测逻辑

脚本会在 `videos/` 目录下查找与视频文件名匹配且包含 `Vocals` 关键词的 WAV 文件：

- 视频：`bandicam 2026-07-12 20-24-44-610.mp4`
- 人声：`1_bandicam 2026-07-12 20-24-44-610_(Vocals).wav`

如果找到人声 WAV → 直接用于识别（精度高）
如果未找到 → fallback 到 FFmpeg 从视频提取原始音频（有配乐干扰）

### 推荐 UVR 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 模型 | UVR-MDX-NET-Inst_HQ_3 | 人声分离质量最高 |
| Segment Size | 256 | 处理分段大小 |
| Overlap | 0.75-0.99 | 越高质量越好，速度越慢 |
| 输出格式 | WAV | 无损 |

## 声纹角色识别技术方案（纯 Embedding 比对）

### 设计思路

**不跑 diarization pipeline，直接对已切分的 WAV 片段提取声纹向量，和预存的角色声纹做余弦相似度比对。**

基础版已经按语句边界切好了独立 WAV 文件，每个文件基本对应一个人说的一句话。
我们只需要回答一个问题："这个片段是谁的声音？" —— 一个向量比对即可。

### 为什么不用 diarization

| 对比项 | 完整 diarization | 纯 embedding 比对 |
|--------|-----------------|-------------------|
| 输入 | 整集完整音频 | 已切好的单句 WAV 片段 |
| 计算内容 | VAD + 分割 + 聚类 + embedding | 仅 embedding 提取 + 余弦相似度 |
| 输出 | 匿名标签（speaker_0, speaker_1） | 直接输出角色名 |
| 是否需要对齐 | 需要和 segments 做时间重叠对齐 | 不需要，一对一关系 |
| 耗时 | 5-8 分钟/集 | 5-10 秒/集 |
| 跨集一致性 | 无（每集重新聚类） | 有（声纹库全局共享） |

### 核心流程

```
┌─────────────────────────────────────────────────────────┐
│  前置条件：extract_dialogue.py 已执行完毕               │
│  output/EP01/ 下已有切分好的 WAV 文件和 segments.json   │
└─────────────────────────────────────────────────────────┘

模式一：声纹注册（--enroll，首次使用，一次性操作）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. 加载 segments.json，展示片段列表（序号 + 文本内容 + 时长）
  2. 用户选择片段并输入角色名
     例：输入角色名 "傻妞"，再输入序号 "3 15 27"
  3. 加载对应 WAV → 3D-Speaker ERes2NetV2 提取 192 维向量
  4. 存入 speaker_profiles/{角色名}.npy
     - 如果该角色已有向量，做增量平均（加权合并）
     - 每个角色建议标注 5-10 个片段，覆盖不同情绪
  5. 更新 profiles.json 元信息
  6. 注册完毕，可直接运行自动识别模式

模式二：自动识别（默认模式，每集几秒）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. 加载 speaker_profiles/ 下所有角色声纹
  2. 加载 segments.json 获取片段列表
  3. 遍历每个 WAV 片段：
     a. 时长 < 1.5 秒 → 标记为"未知"（太短的 embedding 不稳定）
     b. 提取 embedding（~5ms/片段，GPU 下更快）
     c. 计算与每个角色声纹的余弦相似度
     d. 取相似度最高的角色：
        - 最高分 > 阈值 且 与第二名差距 > margin → 确认标注
        - 最高分 > 阈值 但 与第二名差距 ≤ margin → 标记为"不确定"
        - 最高分 ≤ 阈值 → 标记为"未知角色"
  4. 输出 EP01_dialogue_speaker.txt

模式三：补充注册（--enroll-unknown，可选）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. 自动识别后，收集所有"未知角色"片段
  2. 展示这些片段的文本供用户标注
  3. 用户标注后存入声纹库
  4. 适用于新角色出场的情况
```

### 关键算法

#### 余弦相似度

```python
import numpy as np

def cosine_similarity(vec_a, vec_b):
    """计算两个向量的余弦相似度，值域 [-1, 1]"""
    return float(np.dot(vec_a, vec_b) / (np.linalg.norm(vec_a) * np.linalg.norm(vec_b)))
```

#### 增量平均 embedding

```python
def update_profile(existing_embedding, existing_count, new_embedding):
    """加权平均合并新样本，避免单个样本噪声主导"""
    updated = (existing_embedding * existing_count + new_embedding) / (existing_count + 1)
    # 归一化，保持单位向量
    updated = updated / np.linalg.norm(updated)
    return updated, existing_count + 1
```

#### 匹配决策

```python
def identify_speaker(embedding, profiles, threshold=0.65, margin=0.05):
    """
    根据 embedding 判断说话人
    
    返回:
      - (角色名, 置信度)  匹配成功
      - ("不确定", 置信度) top1 和 top2 太接近
      - ("未知角色", 最高分) 所有角色都低于阈值
    """
    scores = {}
    for name, profile_emb in profiles.items():
        scores[name] = cosine_similarity(embedding, profile_emb)
    
    sorted_scores = sorted(scores.items(), key=lambda x: x[1], reverse=True)
    top1_name, top1_score = sorted_scores[0]
    top2_score = sorted_scores[1][1] if len(sorted_scores) > 1 else 0
    
    if top1_score < threshold:
        return "未知角色", top1_score
    if top1_score - top2_score < margin:
        return "不确定", top1_score
    return top1_name, top1_score
```

### 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| embedding 模型 | 3D-Speaker ERes2NetV2 | 输出 192 维向量，中文说话人训练 |
| 相似度阈值 | 0.65 | 低于此值判定为未知角色（需实测调整） |
| top1-top2 差距阈值 | 0.05 | 差距太小时标记为"不确定" |
| 最短片段时长 | 1.5 秒 | 过短的片段跳过，embedding 不稳定 |
| 每角色标注样本数 | 5-10 个 | 覆盖不同情绪状态，取加权平均 |
| 比对方法 | 余弦相似度 | 标准做法，计算量可忽略 |

> 注：阈值 0.65 为初始值。3D-Speaker 在中文人声上的区分度比 pyannote 好，阈值可设更高。实测后根据效果调整。

### 声纹标注策略建议

为提高识别准确率，标注样本时注意：

1. **覆盖不同情绪**：正常对话、生气、低语、喊叫各选 1-2 句
2. **避免背景噪声大的片段**：有明显配乐/音效的跳过
3. **选择时长 2-5 秒的片段**：太短不稳定，太长可能混入其他人
4. **不同集数取样**：避免录音环境偏差，跨集选样更稳健

## 切分策略

FunASR 输出的 sentence 级时间戳基于强制对齐，按语句边界切分。相比 Whisper 的估算时间戳：
- 不会出现尾音"了"被截断的问题
- 句首不会吃掉第一个字
- 切片边界更精准，对后续声纹提取质量也有正面影响

## 已知限制

- **不区分说话人**：基础版不识别"谁在说话"，只输出时间和文本。需运行角色版进行声纹识别。
- **背景噪音**：依赖 UVR GUI 人声分离的质量。如果未做人声分离，配乐干扰会降低识别精度。
- **模型首次下载**：FunASR 模型约 1GB，3D-Speaker 模型约 100MB，首次运行自动下载到 ModelScope 缓存。
- **重叠对话**：两人同时说话时，切分片段可能包含两人声音，声纹比对会标记为"不确定"。

## 文本输出格式

纯文本，一行一句，示例：

```
你在干什么呢
我在想办法回去
你回去干什么
我要找到那个人
```
