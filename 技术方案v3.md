# Cheung-Data 素材处理管线 v3 - GUI 技术方案

## 概述

将 v2 的命令行操作包装为可视化 GUI 界面，使用 PySide6 + QFluentWidgets 实现。核心处理逻辑不变，GUI 仅作为前端壳调用现有的 process.py / enroll.py / export.py 功能。

---

## 技术选型

| 项目 | 选择 | 理由 |
|------|------|------|
| GUI 框架 | PySide6 | Python 生态一致，与现有 funasr/torch 无冲突 |
| UI 组件库 | QFluentWidgets (PyQt-Fluent-Widgets) | Fluent Design 开箱即用，NavigationInterface/ComboBox/ProgressBar/TableWidget 等组件齐全 |
| 音频播放 | PySide6 QMediaPlayer | 内置，无额外依赖 |
| 进程管理 | QProcess | 非阻塞执行命令行，实时捕获输出 |
| 视频播放 | os.startfile() | 用系统默认播放器打开 mp4 |

### 新增依赖

```
PySide6
PyQt-Fluent-Widgets
```

## 顶部选择器

### 命名空间 ComboBox

- 数据源：扫描 `wavs/` 目录下的子文件夹名
- 默认选第一个
- 切换后联动刷新"源文件"下拉框

### 源文件 ComboBox

- 数据源：扫描 `wavs/{namespace}/` 下的 `.wav` 文件，取 stem（去掉扩展名）
- 默认选第一个
- 切换后各页面内容刷新

### 全局共享

所有 4 个页面的操作都基于顶部选中的 namespace 和 wav_stem，不需要每个页面重复选择。

### 数据来源

下拉框数据全部来自目录扫描，不执行命令行：

```python
# 命名空间列表
namespaces = [d.name for d in Path("wavs").iterdir() if d.is_dir()]

# 源文件列表
wavs = [f.stem for f in Path(f"wavs/{ns}").glob("*.wav")]
```

## 页面 3：✏️ 手动修正

### 功能

对 segments.json 中 speaker 为 null 或 cluster 为"过短"的片段，手动指定角色。执行后更新 segments.json 的 speaker 字段并重命名切片文件。

### 界面

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   未标注/过短的片段:                                         │
│   ┌───────────────────────────────────────────────────────┐ │
│   │ ☐ │ #   │ 文件名                  │ 状态  │ 文本│ ▶  │ │
│   │───┼─────┼────────────────────────┼───────┼─────┼─────│ │
│   │ ☑ │ 002 │ 002_00m07s-00m08s_角色2│ 未标注│ 真..│ ▶  │ │
│   │ ☑ │ 005 │ 005_00m25s-00m27s_角色2│ 未标注│ 我..│ ▶  │ │
│   │ ☐ │ 007 │ 007_00m43s-00m44s_过短 │ 过短  │ 小..│ ▶  │ │
│   └───────────────────────────────────────────────────────┘ │
│                                                             │
│   指定角色: [陆小千     ▾]         [▶ 执行标注]              │
│                                                             │
│   可选角色（来自 .npy）: 傻妞、陆小千                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 行为

1. 进入页面时自动加载 `output/{ns}/{wav_stem}/segments.json`
2. 筛选显示：speaker 为 null 且 cluster 不为 null 的条目，或 cluster 为"过短"的条目
3. 角色下拉框数据源：扫描 `speaker_profiles/{ns}/*.npy` 的文件名（去掉 .npy）
4. 用户勾选片段 + 选择角色 → 点击"执行标注"：
   - 更新 segments.json 中被选中片段的 speaker 字段
   - 重命名 clips/ 中对应的文件（用角色名替换原标签）
   - 刷新列表（已标注的从列表中移除）
5. 每行有播放按钮，方便听一下确认是谁
6. 文本列固定宽度，超出显示 `...`，鼠标悬停 tooltip 显示完整文本

### 注意

这是新增功能，不对应现有命令行。它直接操作 segments.json 和文件名，逻辑简单：
- `seg["speaker"] = 选中的角色名`
- `seg["score"] = None`（手动标注无置信度）
- 重命名文件：替换文件名中最后一个 `_` 后面的标签部分

## 底部日志面板

### 设计

- QTextEdit，只读模式
- 实时追加命令输出，每行前加时间戳 `[HH:MM:SS]`
- 右下角"清空"按钮
- 固定高度约 120px，超出可滚动，自动滚到底部
- 不写日志文件，仅在界面显示即可（交互式使用，无需文件持久化）

## 进度条

- 位于日志面板上方
- 执行命令时显示，空闲时隐藏
- 显示当前步骤文字：`███████████░░░ 66% 正在聚类...`

### 进度实现方式

硬编码步骤比例，不是真实文件级进度。每个命令的步骤固定：

**--apply-edit 进度：**

| 步骤 | 进度 |
|------|------|
| 解析 edit.txt | 10% |
| 切分音频 | 40% |
| 提取 embedding | 60% |
| 聚类 | 80% |
| 标注 + 保存 | 100% |

**--gen-edit 进度：**

| 步骤 | 进度 |
|------|------|
| 音频转换 | 10% |
| ASR 识别 | 60% |
| 生成 edit.txt | 80% |
| 生成 reference.mp4 | 100% |

实现：TaskWorker 在调用各步骤前手动 emit 进度 signal，不需要解析 stdout。

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| edit.txt 已存在 | 弹窗提示"edit.txt 已存在，请删除后重新执行" |

## 执行架构

### GUI 调用方式

GUI 不通过 subprocess 调用命令行，而是直接 import 并调用 process.py / enroll.py / export.py 中的函数：

```python
# gui/workers/task_worker.py
from process import gen_edit, apply_edit, label_only, load_config
from enroll import enroll_namespace, load_embedding_model
from export import export_namespace
```

### 异步执行

- 所有耗时操作在 QThread（TaskWorker）中执行
- 通过 signal 将日志文本、进度、完成状态传回主线程
- 主线程负责更新 UI（日志追加、列表刷新、进度条更新）

### stdout 重定向

- TaskWorker 执行前将 `sys.stdout` 重定向到自定义的 SignalStream
- SignalStream 每次 write() 时 emit signal → 主线程 append 到日志面板
- 执行完毕后恢复原始 stdout

## 与 v2 的关系

| 维度 | v2 | v3 |
|------|----|----|
| 交互方式 | 纯命令行 | GUI（命令行保留） |
| 核心逻辑 | process.py / enroll.py / export.py | 不改动，GUI 直接调用 |
| 新增代码 | — | gui.py + gui/ 模块 |
| 新增依赖 | — | PySide6, PyQt-Fluent-Widgets |
| 新增功能 | — | 额外标注（手动指定角色） |

### 对现有代码的改动

几乎不需要改动现有 py 文件。唯一需要调整的：

| 改动 | 原因 | 影响范围 |
|------|------|----------|
| `sys.exit(1)` → `raise Exception(msg)` | GUI 线程中 sys.exit 会杀掉整个程序 | process.py 约 10 处 |
| 无其他改动 | — | — |

print 输出不需要改，通过重定向 stdout 到 SignalStream 即可捕获到日志面板。
